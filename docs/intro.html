fptoken 项目介绍

高频模式挖掘与互斥选择引擎（面向字节词项 / DocID 倒排场景）

Eclipse Project Frequent Itemset Sampling + Full Recompute 452 tests / 0 failed
版本日期：2026-04-22（文档同步）

1. 解决什么问题

  • 在大量文档中识别 高频共现词组（频繁项集）。
  • 从候选中挑选 互斥且收益高 的组合，避免重叠冗余。
  • 在保证质量前提下，通过采样与剪枝降低计算成本。

输入

List<DocTerms>：每条记录是 docId + terms(byte[])

输出

ExclusiveSelectionResult：互斥词组、support、docIds、统计信息

核心入口：`ExclusiveFrequentItemsetSelector`

2. 总体架构

Rows → TermTidsetIndex.buildWithSupportBounds → (Sampling Gate: 可选) Sample Mask + scaled minSupport → BeamFrequentItemsetMiner（候选挖掘） → Full Tidset Recompute（全量回算） → GreedyExclusiveItemsetPicker（互斥筛选） → SelectedGroup + Stats
  • 两阶段索引：先按支持度过滤，再构建位图，减少内存与交集次数。
  • 抽样策略：只在合适规模下启用，最终通过全量回算保证正确性。
  • 互斥挑选：优先长项集和收益，且保证词项不冲突。
关键模块：index / miner / picker 三层解耦

3. 抽样机制（重点）

配置项

  • setSampleRatio(double)
  • setMinSampleCount(int)
  • setSamplingSupportScale(double)

方式：在业务代码启动阶段直接设置

绕过条件

  • sampleRatio <= 0
  • minSupport <= 1
  • docCount < 200
  • sampleSize >= docCount
  • 抽样使用固定随机种子，结果可复现。
  • 采样结果会做 全量回算，避免虚假支持度。
入口类包含 `setSampleRatio/setMinSampleCount/setSamplingSupportScale`

4. 当前质量与测试覆盖

  • 全量默认测试：452 found / 298 started / 298 passed / 0 failed
  • 新增抽样专项：ratio=0、ratio=1、小样本绕过、minSupport=1 绕过、采样确定性。
  • 集成质量对比：采样路径保留主高频模式，覆盖率下降受控。

稳定性修复

并发路径改为 ThreadLocal scratch，避免共享 BitSet 引发越界。

性能优化

OpenHashTable 增加 termId→key O(1) 查找，降低回查成本。

测试：scripts/run-fptoken-tests.ps1 · HTML 报告：scripts/run-fptoken-tests.ps1 -HtmlReport -ExcludePerfTag

5. 如何使用（最简版）

1) 准备 `List<DocTerms>`，每条包含 `docId` 与词项字节数组。

2) 设置参数：`minSupport`、`minItemsetSize`，可选 `maxItemsetSize/maxCandidateCount`。

3) 调用：`selectExclusiveBestItemsetsWithStats(...)`。

4) 读取 `groups + stats`，用于压缩、规则发现或后续策略。

  • 示例代码：`demo/ExclusiveFrequentItemsetSelectorUsageExample`（已含 sample 设置示例）。
  • 可读别名 API：`cn.lxdb.plugins.muqingyu.fptoken.api.MutuallyExclusivePatternSelector`（根包兼容类已移除）。
  • 文档入口：`docs/index.html`。
建议：生产环境统一字节编码（如 UTF-8）并固定参数快照
