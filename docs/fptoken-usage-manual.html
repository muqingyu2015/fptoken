1) Quickstart：5 分钟跑通

步骤 A：准备输入

  • 每条记录是 DocTerms(docId, termRefs)
  • term 用 ByteRef（字节视图）

步骤 B：调用 API

  • ExclusiveFpRowsProcessingApi.processRows(rows)
  • 从 result.getFinalIndexData() 读取三层结果
List<DocTerms> rows = new ArrayList<>();
rows.add(new DocTerms(0, Arrays.asList(ref("A"), ref("B"), ref("C"))));
rows.add(new DocTerms(1, Arrays.asList(ref("A"), ref("B"))));
rows.add(new DocTerms(2, Arrays.asList(ref("D"), ref("E"))));

LineFileProcessingResult result = ExclusiveFpRowsProcessingApi.processRows(rows);

LineFileProcessingResult.FinalIndexData finalData = result.getFinalIndexData();
List<SelectedGroup> groups = finalData.getHighFreqMutexGroupPostings();
List<LineFileProcessingResult.HotTermDocList> hotTerms = finalData.getHighFreqSingleTermPostings();
List<DocTerms> lowHitRows = finalData.getLowHitForwardRows();

你至少能拿到：高频组合层 + 高频单词层 + 低频残差层。

2) 示例 1：基础参数控制

当你要控制支持度、项集长度、候选上限时，优先用 ProcessingOptions。

ExclusiveFpRowsProcessingApi.ProcessingOptions options =
    ExclusiveFpRowsProcessingApi.defaultOptions()
        .withMinSupport(3)
        .withMinItemsetSize(2)
        .withMaxItemsetSize(6)
        .withMaxCandidateCount(200_000)
        .withHotTermThresholdExclusive(4);

LineFileProcessingResult result = ExclusiveFpRowsProcessingApi.processRows(rows, options);
  • minSupport 越大，结果越“保守”，性能通常更稳。
  • maxCandidateCount 是保护阀，防止极端数据打爆耗时。

3) 示例 2：带 n-gram 与 skip-hash 的完整调用

输入是原始行字节时，建议显式设置分词区间与 skip-hash 区间。

ExclusiveFpRowsProcessingApi.ProcessingOptions options =
    ExclusiveFpRowsProcessingApi.defaultOptions()
        .withNgramRange(2, 4)
        .withSkipHashGramRange(2, 6)
        .withMinSupport(80)
        .withMinItemsetSize(2)
        .withHotTermThresholdExclusive(16);

LineFileProcessingResult result = ExclusiveFpRowsProcessingApi.processRows(rawRows, options);

若 rows 已经是“多 term 结构”，请避免重复切词。

4) 示例 3：预合并提示（Pre-merge hints）

典型场景：Lucene merge。先从历史段抽 hints，再用于当前合并段。

// 历史段提取（通常来自上一批 result 的高频层）
List<ExclusiveFpRowsProcessingApi.PremergeHint> mutexHints = ...;
List<ExclusiveFpRowsProcessingApi.PremergeHint> singleHints = ...;

ExclusiveFpRowsProcessingApi.ProcessingOptions hintedOptions =
    baseOptions
        .withPremergeMutexGroupHints(mutexHints)
        .withPremergeSingleTermHints(singleHints)
        .withHintBoostWeight(8)
        .withHintValidationMode(ExclusiveFpRowsProcessingApi.HintValidationMode.FILTER_ONLY);

LineFileProcessingResult hinted = ExclusiveFpRowsProcessingApi.processRows(mergedRows, hintedOptions);
  • 优先使用 mutex hints，single hints 作为补充。
  • 如果 single hints 导致回退，先降权/限流再评估。

5) 示例 4：采样 + 提示 + 效果统计

这是你最关心的“算完就看效果”。

LineFileProcessingResult result = ExclusiveFpRowsProcessingApi.processRows(rows, options);
LineFileProcessingResult.ProcessingStats s = result.getProcessingStats();

System.out.println("candidateCount=" + s.getCandidateCount());
System.out.println("selectedGroups=" + s.getSelectedGroupCount());
System.out.println("selectedGroupTotalEstimatedSaving=" + s.getSelectedGroupTotalEstimatedSaving());
System.out.println("selectedGroupLowSavingPercent=" + s.getSelectedGroupLowSavingPercent());

// 预先提示词效果统计（新增）
System.out.println("premergeHintInputCount=" + s.getPremergeHintInputCount());
System.out.println("mappedHintCandidateCount=" + s.getMappedHintCandidateCount());
System.out.println("mergedDistinctCandidateCount=" + s.getMergedDistinctCandidateCount());
System.out.println("hintMergeMs=" + s.getHintMergeMs());
System.out.println("totalPipelineMs=" + s.getTotalPipelineMs());
System.out.println("hintMergeMsPercent=" + s.getHintMergeMsPercent());

建议上线判据：不仅看组数量，还要看 AvgSavingPerGroup 和 LowSavingGroupPercent。

6) 生产落地建议与排障清单

上线前

  • 先跑 baseline 与 hinted A/B
  • 记录 hint 构建成本与收益成本比
  • 配置好回滚开关（可直接关 hints）

回退判据

  • hintMergeMsPercent 持续升高
  • selectedGroupLowSavingPercent 过高
  • 整体耗时回退但收益不增

推荐流程

  1. 先启用 mutex hints。
  2. 再小流量打开 single hints。
  3. 若回退，先关 single-only，再评估是否关全部 hints。
