# ExclusiveFrequentItemsetSelector 性能测试报告

- 测试目标：按文档数与词数量对比算法耗时
- 算法：高性能贪心互斥选择（byte[] 词）
- 数据：合成数据，包含高频共现 bundle
- 参数：minSupport=3, minItemsetSize=2, maxItemsetSize=4, maxCandidateCount=60000
- 轮次：warmup=1, measure=3

## 结果解读建议

1. 先看同一 vocab 下 docs 增长时 Avg(ms) 的变化，评估文档规模扩展成本。
2. 再看同一 docs 下 vocab 增长时 Avg(ms) 的变化，评估词典规模扩展成本。
3. 若某组 Max(ms) 远高于 Avg(ms)，可适当调低 maxCandidateCount 以换取更稳时延。
4. 合成数据与线上分布可能不同，建议再补一份真实样本基准。
