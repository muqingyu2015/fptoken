2. 潜在缺陷（按优先级）

P0 — 逻辑 / 无法编译

ID位置现象影响
BUG-001 FpGroupHotNgramRebuild.buildFinalHotTerms 迁移旧热词时执行 out.put(e.getKey(), new FPDocList(maxDoc))，丢弃原 hot 中已有 posting。 重建路径下「分析阶段已标 hot」的 doc 列表丢失，除非后续 mergeCommonDocsIntoFinalHot 碰巧覆盖；与设计「保留旧 hot + 合并 common」不符。
BUG-002 FpTokenTermLayout.read_group_id 用 sortableBytesToInt 读 4 字节 group_id，再 强转为 short 返回。 group_id > 32767 时截断/符号错乱，与头部 4 字节 int 布局不一致（设计文档写 group_id 为 int）。
BUG-003 FpFilteredTermsEnum seekCeil 返回类型写成 SeekStatus，未使用 TermsEnum.SeekStatus 或未 import。 全量编译失败（当前仓库独立 javac 已复现）。
BUG-004 全量模块 + lib/lxdb_common-*.jar 缺少补丁 Lucene（BlockTreeTermsWriter.TermsWriter、Terms#iterator_fp、writefp 等）及 slf4j、cn.lucene.proguard.*。 无法在「仅 fptoken 源码 + 单个 common jar」环境完成集成编译；需与完整 LXDB/Lucene 补丁树同构构建。

P1 — 行为风险

ID位置现象建议
BUG-101 FpFilteredTermsEnum#next 原地覆写 rtn.bytes 前 2 字节为 index 前缀。 若底层 TermsEnum 复用同一 BytesRef 缓冲区，可能污染缓存词项；应拷贝或保证迭代器契约。
BUG-102 FpTermKey.viewOf 在 countNewHotNgramsFromCommon 的 HashMap 中 键为切片视图，依赖缓冲区不变；当前循环内每次 new BytesRef 一般安全，但易在后续重构时踩坑。 写入 newhotFreq / TreeMap 时统一 copyOf。
BUG-103 FpGroupDataRebuild#flushto del_term_docid 取自 distinctDocUnion.nextSetBit(0) 后未使用；reuse_bytes 固定 1024、TODO 未实现 dataId 前缀。 删除占位 doc、组头写入可能与设计不一致，需与 Lucene 写路径对齐验收。
BUG-104 测试 / 运行环境 @Tag("lxdb-runtime") 用例在脚本 classpath 下 NoClassDefFoundError（BytesRef 静态初始化）。 在 LXDB IDE/完整依赖下执行 -IncludeLxdbRuntimeTag；或补齐 slf4j 等与 jarloader 一致的依赖。

P2 — 设计缺口 / 维护性

  • BUG-201 FpTokenBlockLevelPolicy.resolveTargetBlockLevel 仅返回 1 或 3，注释中的「级别 2」在策略中未出现。
  • BUG-202 FpGroupDataRebuild#rebuildCommonTermToOrderFromHotDocs 的 Javadoc 仍写「hotTermToOrder」，实现操作的是 commonTermToOrder。
  • BUG-203 设计文档要求对照 FPBlockTreeTermsWriter，源码类名为 FPBlockTreeTermsWriter（大小写混用），集成检索时需注意符号一致。
  • BUG-204 缺少针对 FpGroupHotNgramBitIndex 的 round-trip（flushto → readfrom）与 FpTokenBlockOrchestrator 双路径（original / rebuild）的集成测试（受 BUG-004 阻塞）。

3. 与设计文档的一致性

  • 一致：12 字节 FP 头、组级 hot/common 双表、1~8 byte ngram 桶、8×256 双矩阵位图、交错落盘与 FpBlockInfo 偏移公式（已由单元测试覆盖部分）。
  • 待验证：透传路径 FpGroupDataOriginal + terms.fpBits、iterator_fp 写段编排、查询侧 readfromBanksSelective — 依赖补丁 Lucene，本仓库独立环境未执行。
  • 偏离风险：热词重建丢弃旧 hot doc（BUG-001）与「重建后倒排与位图语义一致」目标冲突。
