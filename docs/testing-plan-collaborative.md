# fptoken 项目测试规划

> 本文档记录各角色对项目测试的规划建议。
> 每个角色从自己的角度分析应该重点测试哪些方面。
> 每个角色独立撰写，汇总在同一文件中。

---

## 1. Developer（开发工程师） — 综合测试规划

### 1.1 测试分层架构

```
单元测试层
  ├── ByteArrayKey 测试（hash/equals/compareTo）
  ├── OpenHashTable 测试（插入/查找/扩容）
  ├── BitSet 操作测试（交集/并集/过滤）
  ├── ByteArrayUtils 测试（hash/compare/copy）
  └── DocTerms 测试（构造/去重/空值）

集成测试层
  ├── TermTidsetIndex 构建测试
  │   ├── build() 单次扫描
  │   ├── buildWithSupportBounds() 过滤构建
  │   └── 结果一致性（与旧版对比）
  ├── BeamFrequentItemsetMiner 挖掘测试
  │   ├── 标准数据挖掘
  │   ├── 大词表挖掘
  │   └── 超时/截断行为
  └── ExclusiveFrequentItemsetSelector 全流程测试
      ├── 全量路径
      ├── 采样路径
      └── 采样 vs 全量一致性

性能测试层
  ├── 索引构建性能（不同规模/配置）
  ├── 挖掘性能（不同词表大小/beam width）
  ├── 采样性能（不同采样比率）
  └── 内存消耗（不同规模）

端到端测试层
  ├── 真实数据场景
  ├── 边界条件
  └── 配置组合测试
```

## 2. Architect（架构师） — 架构测试规划

### 2.1 架构正确性测试

**核心问题：** 架构变更后，输出结果是否与旧版一致？

| 测试场景 | 说明 | 优先级 |
|---|---|---|
| build() vs buildWithSupportBounds 结果一致 | 相同输入下输出相同 | P0 |
| 全量路径 vs 采样路径结果一致 | 采样后的最终输出与全量一致 | P0 |
| 不同配置下结果可重现 | 相同配置 + 相同随机种子 → 相同结果 | P0 |
| 空输入/单文档/边界输入 | 架构的健壮性 | P1 |

### 2.2 架构可扩展性测试

**核心问题：** 架构能否支撑更大规模？

| 测试场景 | 说明 | 优先级 |
|---|---|---|
| 线性扩展测试 | 文档数翻倍，时间是否翻倍？ | P1 |
| 内存峰值测试 | 不同规模下的内存消耗模式 | P1 |
| 分片并行扩展测试 | 核数翻倍，速度是否翻倍？ | P2 |
| 流式处理稳定性 | 持续输入 24 小时不崩溃 | P2 |

### 2.3 组件解耦测试

**核心问题：** 各组件能否独立运行和替换？

| 测试场景 | 说明 | 优先级 |
|---|---|---|
| 索引构建独立测试 | 不依赖挖掘逻辑 | P1 |
| 挖掘独立测试 | 可使用预构建索引 | P1 |
| 采样独立测试 | 可插入不同采样策略 | P1 |
| 编码器独立测试 | 可插入不同编码方案 | P2 |

## 3. Java Perf Guru（Java 性能专家） — 性能测试规划

### 3.1 性能基准测试

**核心问题：** 每次优化后，性能是变好了还是变差了？

#### 测试矩阵

| 文档数 | 词表大小 | 每文档词数 | minSupport | 说明 |
|---|---|---|---|---|
| 100 | 100 | 10 | 5 | 小数据基线 |
| 1K | 1K | 20 | 50 | 中等数据 |
| 10K | 5K | 30 | 500 | 典型场景 |
| 50K | 20K | 50 | 1000 | 大词表 |
| 100K | 50K | 50 | 5000 | 最大规模 |
| 100K | 100K | 30 | 5000 | 超大词表（重尾） |

#### 性能指标

| 指标 | 说明 | 阈值 |
|---|---|---|
| 索引构建时间 | 从输入到索引完成 | < 基线 110% |
| 挖掘时间 | 从索引到挖掘完成 | < 基线 110% |
| 总时间 | 端到端 | < 基线 110% |
| 内存峰值 | Runtime.totalMemory() 峰值 | < 基线 120% |
| GC 时间 | GC 日志分析 | < 基线 120% |

### 3.2 GC 性能测试

**核心问题：** 优化是否引入了额外的 GC 压力？

| 测试 | 说明 | 优先级 |
|---|---|---|
| GC 频率测试 | 不同规模下的 GC 次数 | P1 |
| GC 停顿时间测试 | Full GC 最长停顿 | P1 |
| 对象分配率测试 | 每秒分配字节数 | P2 |
| 内存泄漏测试 | 长时间运行后内存是否稳定 | P1 |

### 3.3 JVM 参数测试

| 参数组合 | 说明 | 优先级 |
|---|---|---|
| -Xms512m -Xmx512m | 小堆场景 | P1 |
| -Xms1g -Xmx1g | 默认堆 | P1 |
| -Xms4g -Xmx4g | 大堆场景 | P1 |
| -XX:+UseG1GC | G1GC 对比 | P1 |
| -XX:+UseParallelGC | ParallelGC 对比 | P2 |

## 4. QA Lead（质量保证负责人） — 质量测试规划

### 4.1 功能正确性测试

#### 核心功能测试

| 测试用例 | 预期结果 | 优先级 |
|---|---|---|
| 空文档列表 | 返回空结果 | P0 |
| 单文档单词 | 正确挖掘 | P0 |
| 所有文档都相同 | 一个包含所有词的组合 | P0 |
| 所有文档都不同 | 空结果（无频繁项） | P0 |
| docId 不连续 | 正确处理 | P1 |
| docId 重复 | 报错或去重 | P1 |
| term 为空/重复 | 自动过滤 | P1 |

#### 配置参数边界测试

| 参数 | 边界值 | 预期 | 优先级 |
|---|---|---|---|
| minSupport | 1 | 不过滤 | P0 |
| minSupport | docCount | 只包含全覆盖词 | P0 |
| minSupport | docCount + 1 | 空结果 | P0 |
| minItemsetSize | 1 | 单词组合 | P1 |
| minItemsetSize | > docCount | 空结果 | P1 |
| maxItemsetSize | 1 | 不挖掘 | P1 |
| sampleRatio | 0.0 | 跳过采样 | P1 |
| sampleRatio | 1.0 | 全量挖掘 | P1 |
| samplingSupportScale | 0.0 | 自动缩放 | P1 |
| samplingSupportScale | 1.0 | 不缩放 | P1 |

### 4.2 采样质量测试

#### 核心指标

| 指标 | 阈值 | 说明 |
|---|---|---|
| Groups diff | = 0 | 组合数量必须一致 |
| Top-10 doc-refs diff | = 0 | 前 10 组合的 doc-refs 必须一致 |
| 全量 doc-refs diff | < 1% | 全部 doc-refs 差异小于 1% |
| 高频组合召回率 | > 99% | 全量结果中支持度 > minSupport×2 的组合 |

#### 测试场景

| 场景 | 说明 | 优先级 |
|---|---|---|
| 均匀分布 | 词均匀分布 | P0 |
| Zipf 分布 | 少数高频词 + 大量低频词 | P0 |
| 偏斜分布 | 某些文档密集某些稀疏 | P1 |
| 极小数据集 | < 50 文档 | P1 |
| 超大词表 | 唯一词数 > 文档数 × 10 | P1 |

### 4.3 回归测试

**每次代码变更后需要运行：**
1. 功能正确性测试（全部）
2. 采样质量测试（主要场景）
3. 性能基准测试（对比基线）
4. 配置参数边界测试（变更相关）

### 4.4 膨胀率测试

| 指标 | 说明 | 优先级 |
|---|---|---|
| 原始索引大小 | 不压缩的大小 | P1 |
| 压缩后索引大小 | 应用编码后的大小 | P1 |
| 膨胀率 | 压缩后/原始 | P1 |
| 组合覆盖率 | 被编码的查询比例 | P2 |
