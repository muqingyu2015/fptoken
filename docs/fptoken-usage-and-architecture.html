FPToken 设计原理与使用指南

适用版本：当前 Eclipse 项目版 | 场景重点：高频模式挖掘、互斥筛选、单线程稳定运行

频繁项集 高频优先 抽样加速 两阶段索引 互斥选择

1. 设计思路

  1. 目标导向：你要的是高频、可压缩、可落地的模式，而不是“穷举所有可能组合”。
  2. 先缩空间再计算：通过支持度过滤、覆盖率上限和候选上限，把搜索空间压到可控范围。
  3. 先挖掘再决策：先产出候选项集，再做互斥筛选，保证最终结果互不重叠、可执行。
  4. 近似但可控：Beam Search + 两阶段 picker，追求工程效率，同时保留关键统计指标（候选数、截断标记、交集次数）。
  5. 可配置：抽样比例、最小样本数、支持度缩放均通过代码启动阶段设置。
当前实现非常适合“高频优先 + 单线程稳定输出”的业务诉求。

2. 核心原理

2.1 两阶段索引构建（TermTidsetIndex）

  • 第一阶段统计词频，过滤掉低频词和过热词（可按 maxDocCoverageRatio 控制）。
  • 第二阶段仅为保留词构建位图（BitSet），减少内存峰值和无效交集计算。

2.2 高频抽样路径（可选）

  • 先抽样文档做候选挖掘，再回到全量数据做支持度回算。
  • 支持度阈值可按采样比例自动缩放：sampledMinSupport = minSupport * scale。
  • 当样本太小、比例为 0、或其他边界条件触发时，会自动回退全量路径。

2.3 候选挖掘（BeamFrequentItemsetMiner）

  • 按支持度筛选频繁词，逐层扩展前缀，使用 Beam Width 和分支上限控制爆炸。
  • 用位图交集计算支持度，按分数保留 top-k 前缀和 top-k 候选。
  • 支持超时和 idle-level 提前停止，避免长尾计算。

2.4 互斥选择（TwoPhaseExclusiveItemsetPicker）

  • 第一阶段贪心构造可行解；第二阶段在 swap 预算内做局部替换优化。
  • 确保最终选中项集两两互斥（不共享 termId）。

4. 使用方法

4.1 Java 代码调用（推荐）

import cn.lxdb.plugins.muqingyu.fptoken.ExclusiveFrequentItemsetSelector;
import cn.lxdb.plugins.muqingyu.fptoken.model.DocTerms;
import cn.lxdb.plugins.muqingyu.fptoken.model.ExclusiveSelectionResult;
import java.util.List;

List<DocTerms> rows = ...; // 你的文档-词项数据
ExclusiveSelectionResult result =
    ExclusiveFrequentItemsetSelector.selectExclusiveBestItemsetsWithStats(
        rows,
        100,   // minSupport
        2,     // minItemsetSize
        6,     // maxItemsetSize
        200000 // maxCandidateCount
    );

System.out.println("groups=" + result.getGroups().size());
System.out.println("candidates=" + result.getCandidateCount());
System.out.println("frequentTerms=" + result.getFrequentTermCount());
System.out.println("truncated=" + result.isTruncatedByCandidateLimit());

可读别名 API（用于语义化命名）位于 cn.lxdb.plugins.muqingyu.fptoken.api.MutuallyExclusivePatternSelector，根包兼容类已移除。

4.2 抽样参数配置

当前版本采用代码内配置（不再读取 properties 文件）：

ExclusiveFrequentItemsetSelector.setSamplingEnabled(true);
ExclusiveFrequentItemsetSelector.setSampleRatio(0.30d);
ExclusiveFrequentItemsetSelector.setMinSampleCount(50);
ExclusiveFrequentItemsetSelector.setSamplingSupportScale(0.0d); // 自动按采样比例缩放
参数 含义 建议
sampleRatio 抽样比例（0~1） 高频目标建议 0.2~0.4；设为 0 则走全量路径
minSampleCount 最小采样文档数 建议 50~200，根据数据规模调整
supportScale 采样支持度缩放系数 0 表示自动按样本比例缩放，通常最稳
如果你明确不要抽样，可以在启动时将 sampleRatio=0 或在代码中关闭 sampling 开关。
