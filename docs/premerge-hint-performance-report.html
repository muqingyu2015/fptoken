FPToken 预合并提示（Pre-merge Hints）性能对比报告

报告类型：模拟 Lucene 合并场景的合成负载 · 指标为单次 JVM 内多次测量的中位数耗时（ms）
数据来源：MergePremergeHintAccelerationPerformanceTest（需 -Dfptoken.runPerfTests=true）
生成说明：数值会随机器负载波动；下列为一次代表性运行的日志摘录，用于说明量级与对比关系。

结论摘要

  • 仅高频提示（mutex + single）相对无提示基线：本次运行中位数由 13 ms → 7 ms，约 −46% 耗时（提升约 86% 吞吐等价表述：同时间可跑约 1.86 倍次数）。
  • 拆分提示来源（仅 mutex / 仅 single / 二者）：本次运行基线 10 ms；both=9 ms 略优；mutex-only 在个别运行中可能略慢于基线（毫秒级抖动），测试用例已对回归做宽松阈值保护。
  • 实现说明：merge 路径使用 withPremergeMutexGroupHints 与 withPremergeSingleTermHints 分开传入，并配合 hintBoostWeight；不再通过 ProcessingOptions 传入历史低频正排先验。

3. 如何复现

在项目根目录执行（PowerShell）：

./scripts/run-fptoken-tests.ps1 -Perf

或仅跑本报告相关类：

java --% -Dfptoken.runPerfTests=true -jar lib/junit-platform-console-standalone-1.10.2.jar execute --class-path "bin;bin-test" --select-class cn.lxdb.plugins.muqingyu.fptoken.tests.performance.MergePremergeHintAccelerationPerformanceTest

首次运行前需已通过 ./scripts/run-fptoken-tests.ps1 完成编译，或确保 bin / bin-test 与 lib/junit-platform-console-standalone-1.10.2.jar 存在。生成 HTML 测试汇总：./scripts/run-fptoken-tests.ps1 -HtmlReport -ExcludePerfTag（或 ./scripts/run-tests-html-report.ps1）。

4. 重要说明（读报告必看）

  • 上述数字来自合成 merge-like 数据集，用于回归与趋势验证，不等价于生产 Lucene 合并的绝对毫秒数。
  • 当 hint 与当前段重叠度低、或提示扩大候选池导致回算成本上升时，收益可能不明显甚至略慢；主测试会打印 reason= 辅助诊断。
  • 输出中的 lowHitForwardRows 仍是本批次的索引结果层（供 skip 等使用），与 merge 输入侧的 pre-merge 选项无关。
