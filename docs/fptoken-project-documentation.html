1. 项目说明

fptoken 是一个面向二进制/字节流场景的频繁模式发现与互斥筛选模块。它将输入的文档词项 （DocTerms）转换为垂直索引（词到文档位图），再进行频繁项集挖掘，最后输出词级互斥的候选组， 供检索优化、索引压缩或规则抽取使用。

  • 输入：List<DocTerms>（每条文档包含 docId 和词字节数组集合）
  • 输出：ExclusiveSelectionResult（组列表 + 统计信息）
  • 核心能力：频繁项集近似挖掘 + 词级互斥优化 + 统计可观测性

2. 项目背景

从项目 README 可知，目标是改善二进制检索中常见方案（如 n-gram、Data Skipping Index）的缺陷： 控制索引膨胀、提升高命中词条检索稳定性，并兼顾低频场景效率。

  • 传统 n-gram 在大规模数据下存储膨胀显著。
  • Data Skipping 在高命中词条时易退化。
  • 本项目采用“频繁项集思想 + 互斥筛选”，通过模式结构化提升整体性价比。
定位上，它不是单纯挖掘“最多模式”，而是输出更可执行、更去重的“高价值互斥模式集合”。

5. 实现原理

5.1 垂直索引原理

  • 把“文档视角”转为“词视角”：每个词对应一个文档位图（BitSet）。
  • 项集支持度可通过位图交集后基数计算：support(X∪Y)=cardinality(T(X)&T(Y))。
  • 相比逐文档扫描，交集路径更适合高频模式迭代扩展。

5.2 Beam 挖掘原理

  • 先收集频繁 1-项，再逐层扩展前缀。
  • 每层仅保留 top-K 前缀（Beam），控制组合爆炸。
  • 支持早停：候选上限、空层、可选超时与 idle level。

5.3 互斥筛选原理

  • 词级互斥：最终选择结果中的任意两个项集不能共享同一 termId。
  • 第一阶段贪心按长度/收益/支持度排序快速拿到可行解。
  • 第二阶段在交换预算内做 1-opt 替换，提升目标值但不破坏互斥约束。
该方案偏工程近似最优，目标是稳定和可控；不是全局最优求解器。

6. 重要实现与原理

实现点原理说明工程价值
TermTidsetIndex.build 为每个唯一词分配连续 termId，维护 idToTerm 与 tidsetsByTermId 双结构。 后续挖掘可 O(1) 取位图，termId 连续便于数组化高效访问。
BeamFrequentItemsetMiner.appendCandidate 写入候选前后都检查 maxCandidateCount，命中即标记 truncatedByCandidateLimit。 防止候选爆炸引发不可控资源消耗。
GreedyExclusiveItemsetPicker 位图标记已占用 termId，线性扫描冲突检测。 复杂度可控，作为低成本稳定基线。
TwoPhaseExclusiveItemsetPicker 以 Greedy 结果为初解，按 objective 做挑战者替换（1-opt）。 在有限预算下提升解质量，保持线上可预测延迟。
DTO 防御性复制 模型对象返回副本或不可变容器，避免外部误改内部状态。 提高 API 契约稳定性，降低调用方副作用风险。

7. 类调用关系与原理

7.1 核心调用关系图

7.2 关键调用链解释

  1. Facade 收到请求后做空输入短路与配置构建。
  2. Index 层把原始词映射成 termId，并构建每个词的文档位图。
  3. Miner 在位图上进行 Beam 扩展，输出候选及统计字段。
  4. Picker 对候选进行词级互斥筛选，得到最终候选子集。
  5. Facade 完成 termId 还原、docId 展开、统计封装并返回结果。
